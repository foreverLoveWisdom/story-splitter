# INVEST Criteria

## The One Thing

Good user stories are **Independent, Negotiable, Valuable, Estimable, Small, and Testable**.

## The 6 Criteria

**I - Independent**
- Can be developed in any order
- No tight dependencies on other stories
- Team can parallelize work

**N - Negotiable**
- Details can be discussed and adjusted
- Not a rigid contract
- Focus on outcome, not implementation

**V - Valuable**
- Delivers value to end user or business
- Can explain "So that [why]" clearly
- Not just technical tasks

**E - Estimable**
- Team can estimate effort/complexity
- Requirements are clear enough
- Not too vague or enormous

**S - Small**
- Fits in one sprint
- 3-8 story points maximum
- Can be completed in days, not weeks

**T - Testable**
- Clear acceptance criteria
- Can verify when done
- QA can test without guessing

## Checklist

Use after breaking down stories:

- [ ] Independent - Can develop in any order?
- [ ] Negotiable - Open to discussion?
- [ ] Valuable - User/business benefit clear?
- [ ] Estimable - Team can estimate?
- [ ] Small - Fits in one sprint?
- [ ] Testable - Clear success criteria?

## AI Collaboration Context

After using story-splitter prompts, run this checklist:
- Stories violating INVEST = need further breakdown
- Use story breakdown prompt again if stories too large (not Small)
- Add ACs if not Testable
